# Lumark v0.1 디자인 프롬프트 모음

> **용도**: Claude Design (아티팩트 기반 UI 시안 생성)에 붙여넣기 위한 프롬프트.
> **톤**: 클래식하면서 세련되고 현대적. 브라운 계열, 전문적인 느낌.
> **순서**: 0번(디자인 시스템)을 먼저 한 번 만들고, 그걸 1·2·3번 프롬프트에 함께 첨부해서 일관성 유지.

---

## 0. 디자인 시스템 (먼저 만들고 시안마다 첨부)

### 프롬프트

```
나는 iOS 학습 도구 앱 "Lumark"의 디자인 시스템을 만들고 있어.
한 줄 요약: "형광펜만 그으면, 정리 노트가 알아서 쌓이는 iOS 앱"
타겟 사용자: 간호학과 학생.

핵심 컨셉:
- 사용자가 굿노트 PDF에 그은 4가지 형광펜 색(노랑/주황/분홍/파랑)을 자동 인식해서 마크다운 노트로 변환
- 콘텐츠 자체가 이미 강한 채도의 형광펜 색깔을 가지고 있음
- 따라서 UI 크롬은 콘텐츠와 싸우지 않도록 차분해야 함

비주얼 무드:
- 책상 위에 놓인 가죽 노트 + 만년필 잉크 같은 무게감
- "iA Writer" "Bear" "Things 3" 같은 사색적·전문적 도구의 절제된 미감
- 한국 출판물의 정갈한 활자 감각 (Noto Sans KR / Pretendard 베이스)
- 너무 빈티지하면 촌스럽고, 너무 미니멀하면 차가움 → 그 사이의 따뜻한 모더니즘

다음 항목을 SwiftUI에서 바로 옮길 수 있는 형태로 정의해줘 (HTML/CSS 컴포넌트 라이브러리 페이지로 시각화):

1. 컬러 팔레트 (Light / Dark 둘 다)
   - Primary Brown: 가죽·원목 느낌의 따뜻한 갈색 (메인 액션·헤더)
   - Background Cream: 종이 같은 따뜻한 베이지 (메인 배경)
   - Surface: 카드·시트 배경 (Cream보다 한 단계 옅거나 짙음)
   - Ink: 본문 글자색 (검정 대신 진한 갈색·다크 에스프레소)
   - Subtle: 보조 텍스트·아이콘
   - Divider: 구분선 (거의 안 보일 정도로 미세)
   - Brass Accent: 황동 색 (Pro 느낌의 미세한 액센트, 아이콘 강조 등 제한적 사용)

2. 형광펜 4색 (콘텐츠 색, 절대 바꾸지 말 것 — 단 채도만 살짝 정돈)
   - Yellow (핵심): 노트 형광펜 노랑
   - Orange (주제): 노트 형광펜 주황
   - Pink (분홍): 노트 형광펜 분홍
   - Blue (파랑): 노트 형광펜 파랑
   - 각 색은 토글 칩, 색상 필터, 색별 본문 강조에 동일하게 재사용됨
   - Light/Dark 모드에서 모두 가독성 유지하는 배경+테두리 변형까지 함께 제시

3. 타이포그래피
   - 한글 본문: Pretendard (또는 Noto Sans KR)
   - 한글 강조/타이틀: 살짝 세리프 무드가 들어간 옵션도 같이 (Nanum Myeongjo 등)
   - 사이즈 스케일: Display / H1 / H2 / Body / Caption / Mono (코드)
   - Line-height와 letter-spacing까지 명시

4. 모서리·그림자·간격
   - Radius: sm(6) / md(12) / lg(20) — 살짝 부드럽지만 너무 둥글지 않음
   - Shadow: 거의 안 쓰되, 카드 들어올림용 1단계만
   - Spacing scale: 4 / 8 / 12 / 16 / 24 / 32 / 48

5. 컴포넌트 미리보기
   - Primary Button / Secondary Button / Ghost Button
   - 토글 칩 (4색 형광펜용)
   - 카드 (최근 작업용)
   - 탭 토글 (마크다운 ↔ PDF)
   - 하단 액션 바
   - 진행률 바 (얇고 우아하게)
   - Empty State 일러스트레이션 가이드라인 (선화 스타일, 갈색 단색)

출력은 한 페이지 짜리 디자인 시스템 가이드 형태로,
좌측 사이드 네비게이션 + 우측 본문 패널 레이아웃.
모바일(375px)과 데스크탑(1280px) 모두에서 깨지지 않도록.
```

### 결과물 활용

- 이 디자인 시스템 페이지를 만들고 나서, **컬러 토큰·폰트·간격값을 SwiftUI Color/Font extension으로 옮겨두면** 코딩 속도가 가장 많이 줄어듦.
- 1·2·3번 프롬프트 머리에 "0번에서 정의한 디자인 시스템을 따른다"고 명시.

## 1. ResultView (최우선)

### 왜 우선순위 1

- 가장 복잡한 화면. 마크다운 ↔ PDF 토글, 4색 필터 칩, 하단 액션 바가 한 화면에 어우러져야 함.
- 사용자가 "어, 이거 잘 만들었네"를 처음 느끼는 지점.
- 여기 시안 1장이 코딩 속도를 가장 많이 줄여줌.

### 프롬프트

```
[0번 디자인 시스템을 첨부하거나 위 컬러·타이포 요약을 같이 붙임]

위 디자인 시스템을 따라 iOS 앱 Lumark의 ResultView 화면 시안을 만들어줘.
모바일 사이즈(iPhone 15 기준 393x852), Light 모드와 Dark 모드 둘 다.

화면 구성 (위→아래):

[상단 네비게이션 바]
- 왼쪽: 뒤로가기 아이콘 (chevron.left)
- 가운데: 노트 제목 ("항생제정리"). 길면 말줄임.
- 오른쪽: 더보기 메뉴 (ellipsis)
- 배경은 Cream보다 살짝 옅게, 하단에 거의 안 보일 정도의 divider

[탭 토글] (Primary 영역, 상단 네비 바로 아래)
- 두 개의 탭: "마크다운"  |  "원본 PDF"
- 현재 선택: "마크다운"
- 선택된 탭은 Primary Brown underline + Ink 색 텍스트
- 선택 안 된 탭은 Subtle 색
- iOS Segmented Control이 아니라 더 우아한 underline tab 스타일

[색상 필터 칩 영역] (탭 아래, 좌우 스크롤 가능)
- 4개 칩: "노랑 (핵심)"  "주황 (주제)"  "분홍"  "파랑"
- 각 칩 왼쪽에 동그란 색 dot
- 활성/비활성 상태 둘 다 보여줘
  - 활성: 해당 색의 옅은 배경 + Ink 텍스트 + 색 dot 진하게
  - 비활성: 투명 배경 + Subtle 텍스트 + 색 dot 옅게
- 노랑·주황은 활성 ON, 분홍·파랑은 비활성 OFF (기본값)

[본문 영역] (마크다운 미리보기, 스크롤)
디자인 시스템의 타이포 위계대로 마크다운을 자연스럽게 렌더링.
예시 콘텐츠:

  # 항생제정리

  ## 항생제의 분류
  - 베타락탐계는 세포벽 합성을 억제
  - 페니실린 알레르기 환자 주의
  - 세팔로스포린은 1~5세대까지

  ## 부작용 모니터링
  - 신독성 신호 — BUN/Cr 상승
  - 청신경 독성 — 가역적
  - 위막성 대장염 — 클로스트리디움 디피실

  ---
  ### 추가 메모
  **보충 (분홍)**
  - 분홍 펜으로 표시된 항목들

핵심: 본문 글머리표 옆에 그 항목이 어떤 색 형광펜에서 왔는지
**왼쪽 세로 컬러 바(2px)** 로 표시. 노랑 항목이면 노랑 바, 주황 항목이면 주황 바.
이게 Lumark의 핵심 시그니처 디테일. 절대 빼지 말 것.

[하단 액션 바] (Safe area 바로 위, 고정)
- 좌→우 3개 버튼, 동일 비중
- [복사]  [공유]  [PDF 내보내기]
- 각 버튼은 아이콘(SF Symbols 스타일) + 작은 라벨
- 배경은 Surface 색에 상단 미세 divider
- 가장 오른쪽 "PDF 내보내기"가 Primary Brown filled (강조), 나머지 둘은 Ghost

전체 톤:
- 책의 한 페이지를 보는 듯한 여백·호흡
- 형광펜 색은 살아있되 UI 크롬은 조용함
- 한글 폰트 가독성 최우선 (line-height 충분히)

같은 화면을 두 가지 변형으로 그려줘:
(A) 마크다운 탭이 선택된 상태 (위 설명대로)
(B) "원본 PDF" 탭이 선택된 상태 — PDF 페이지가 보이고 그 위에
    형광펜 영역이 살짝 컬러 오버레이된 모습. 색상 칩은 여전히 보이고
    필터링에 따라 오버레이도 켜지고 꺼짐.

Light · Dark · A탭 · B탭 = 총 4개 패널을 한 화면에 나란히.
```

## 2. HomeView

### 왜 우선순위 2

- 첫인상 + 브랜드 톤이 박히는 곳.
- "마찰 0"이라는 가치가 시각적으로 표현되어야 함 → 액션이 한 눈에 들어와야 함.
- 최근 작업 카드가 누적되며 "내 학기 정리본"이 쌓이는 정서적 만족감을 줘야 함.

### 프롬프트

```
[0번 디자인 시스템 첨부]

위 디자인 시스템을 따라 iOS 앱 Lumark의 HomeView 화면 시안을 만들어줘.
모바일 사이즈(393x852), Light 모드와 Dark 모드 둘 다.

화면 구성 (위→아래):

[헤더]
- 좌측 상단에 "Lumark" 워드마크 (디자인 시스템의 강조 폰트 사용)
- 그 아래 부제 한 줄, Subtle 색:
  "형광펜만 그으면, 정리 노트가 알아서 쌓여요"
- 우측 상단: 설정 아이콘 (gearshape)

[주요 액션 카드] (헤더 아래, 가장 큰 시각 비중)
좌우 2x2 그리드, 4개 카드. 각 카드는 큰 라운드 모서리 + 미세한 그림자.

  [업로드]              [카메라]
  PDF·이미지 선택        직접 촬영

  [최근 작업]            [설정]
  내 정리본              색·라벨

- 각 카드는 큼직한 아이콘(상단) + 라벨(중간) + 짧은 설명(하단)
- 업로드·카메라는 Primary Brown filled — 사용자가 가장 자주 누름
- 최근 작업·설정은 Surface 배경 + Ink 텍스트
- 카드 비율: 정사각형 또는 살짝 세로 긴 1:1.1 정도

[안내 배너] (카드 아래)
- 한 줄 배너 카드, 미세한 점선 테두리
- 왼쪽 아이콘 + 텍스트: "굿노트에서 공유 → Lumark로 보내면 자동으로 받아요"
- 우측 chevron, 누르면 도움말로 이동
- 너무 크지 않게, 한 번 보면 충분한 정도의 시각 비중

[최근 작업 섹션]
- 섹션 헤더: "최근 작업" + 우측에 "모두 보기"
- 그 아래 카드 리스트 3~4개

  각 카드 한 줄:
  - 좌측: 작은 PDF 썸네일 또는 페이지 수 칩 ("4p")
  - 가운데: 노트 제목 (Ink, Body) + 변환 날짜 (Subtle, Caption)
  - 가운데 아래: 사용된 색깔 dot 배지 (예: 노랑 dot · 주황 dot · 분홍 dot)
  - 우측: chevron

- 카드 사이 간격은 12pt, divider 없음

[Empty State 변형도 같이]
"아직 변환한 노트가 없어요" 메시지 + 갈색 단색 선화 일러스트 (책+형광펜)
+ "굿노트 PDF를 공유로 보내보세요" CTA

전체 톤:
- 책상 위에 잘 정돈된 가죽 노트를 펼친 느낌
- 정보보다 호흡과 여백이 우선
- 형광펜 색은 최근 작업 카드의 dot 배지에서만 살짝 등장 (티저)

같은 화면을 두 가지 변형으로:
(A) 최근 작업 3개가 있는 정상 상태
(B) Empty State (첫 실행 직후)

Light · Dark · A · B = 4개 패널 나란히.
```

## 3. 앱 아이콘 + 로고

### 왜 우선순위 3

- 한 번 정하면 모든 화면·앱스토어·공유 카드에 반복됨.
- 4색 형광펜이 곧 정체성인 앱이라 **아이콘 안에 4색이 들어가야 인지도가 만들어짐**.
- 다만 클래식·전문성을 추구하므로 4색은 작게, 무게는 갈색이 잡음.

### 프롬프트

```
[0번 디자인 시스템 첨부]

iOS 앱 Lumark의 앱 아이콘과 워드마크 로고를 만들어줘.

브랜드 컨셉:
- 이름 어원: Lumin(빛) + Mark(표시) = 형광펜으로 표시한 자리에 빛을 비춘다
- 한 줄 요약: 형광펜만 그으면 정리 노트가 알아서 쌓이는 학습 도구
- 정체성: 4가지 색(노랑/주황/분홍/파랑) 형광펜이 곧 의미
- 톤: 클래식하면서 세련되고 현대적, 가죽 노트+잉크 같은 무게감의 브라운 베이스

요청 1: 앱 아이콘 (1024x1024 정사각형, iOS 라운드 마스크 적용 후 미리보기까지)

방향 A: 활자 베이스
- 진한 가죽 브라운 배경
- 가운데 큰 모노그램 "L" — 살짝 세리프가 들어간 현대 활자
- L 아래쪽 baseline에 4색 작은 점 4개가 가로로 (노/주/분/파) — 형광펜 도트
- 황동 액센트 미세하게

방향 B: 펜 끝
- 진한 가죽 브라운 배경
- 가운데 형광펜 끝(chisel tip)이 사선으로 살짝 보이고
- 그 끝에서 4색 빛줄기(짧은 stroke)가 부드럽게 번지는 모습
- 매우 정제된 일러스트, 디테일 최소

방향 C: 추상 마크
- Cream 배경
- 가운데에 책갈피 모양의 갈색 띠
- 띠 위에 4색 작은 dot row
- 가장 미니멀

세 방향 모두 시안으로 보여줘. 각 시안은 1024 풀사이즈 + 180(홈스크린 실제 크기) 미리보기.
검정·흰색 배경 위에 올렸을 때 대비도 확인되도록.

요청 2: 워드마크 로고 (가로형, 헤더용)
- "Lumark" 글자만
- 위에서 정한 강조 폰트 변형 (살짝 세리프 무드 + 현대적 비율)
- m 자의 mark, k 자의 마크 부분에 미세한 디자인 요소 (예: 점 하나, stroke 하나)
```

## 부록. 굳이 디자인 안 해와도 되는 화면 (이유 포함)

- **ShareView (Share Extension UI)**: 시스템 제약이 많아 디자인 자유도 낮음. v0.1에서는 표준 컴포넌트 + 색만 맞춤. 나중에 고도화.

## 운영 노트

- **순서 권장**: 0번 → 3번(아이콘·워드마크) → 1번(ResultView) → 2번(HomeView).
  로고·워드마크가 먼저 정해져야 HomeView 헤더가 흔들리지 않음.
- **픽셀 완성도 추구 금지**: 스펙 §7의 Day 2~4 기술 검증 게이트가 RED 나면 색 검출 자체가 안 될 수 있어서 UI 시안은 의미가 줄어듦. **와이어프레임 수준 + 무드 보드** 정도면 충분.
- **디자인은 코딩과 병렬로**: 게이트 검증 중에도 디자인 시안은 계속 다듬을 수 있음. 단, 시안을 "최종"으로 못 박지 말고 코드와 실제 데이터를 본 뒤 한 번 더 조정한다는 전제로 진행.
- **시안 → 코드 옮길 때**: 컬러·폰트·간격 토큰을 `Theme.swift` 한 파일에 박아두고, 모든 View가 거기서만 참조하게 하면 톤 변경 비용이 0에 수렴.
