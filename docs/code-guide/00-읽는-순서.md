# Lumark 코드 구조 — 중학생 버전 가이드

> 코딩 모르는 친구한테 "내가 만든 이 앱이 어떻게 굴러가"를 설명할 때 같이 보면 좋은 문서.
> 비유 위주, 표 위주, 그림 위주.

---

## 읽는 순서

1. **`01-전체구조-한눈에.md`**
   앱이 뭐 하는지, 부서가 어떻게 나뉘었는지. **5분이면 읽어.**

2. **`02-폴더별-자세히.md`**
   각 폴더 → 각 파일이 한 줄로 뭘 하는지. 사전처럼 옆에 두고 봐도 됨.

3. **`03-데이터-흐름.md`**
   "PDF가 들어와서 마크다운이 나가기까지"를 컨베이어 벨트로. 가장 재밌는 편.

---

## 한 문장 요약

> **Lumark는 형광펜으로 그은 PDF를 자동으로 마크다운 노트로 정리해주는 iOS 앱이고,
> 코드는 공장처럼 부서가 나뉘어 있어서 (정문/일꾼/진열대/별관),
> PDF가 들어오면 6단계 컨베이어 벨트를 거쳐 마크다운으로 나간다.**

---

## 누가 보면 좋을까

- **코딩 안 해본 친구**: 1편만 봐도 "아, 대충 이런 앱이구나" 이해 가능.
- **Swift는 모르지만 다른 언어 해본 사람**: 2편까지 보면 어디부터 코드 까볼지 감 잡힘.
- **나중에 내가 다시 볼 때**: 3편이 가장 유용. "왜 이렇게 짰지?"가 다 들어 있음.
